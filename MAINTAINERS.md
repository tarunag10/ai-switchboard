# Maintainers

## Project Owner

- Tarun Agarwal (`@tarunag10`)

Tarun Agarwal is the owner, release authority, and final approver for this
repository.

## Approval Policy

No pull request from another person, bot, fork, dependency-update service, or
external contributor should be accepted or merged unless Tarun Agarwal explicitly
approves it.

Passing CI is required for many changes, but passing CI is not approval.

## Maintainer Responsibilities

The maintainer is responsible for:

- Reviewing and approving incoming pull requests.
- Protecting signing keys, updater keys, API credentials, and release secrets.
- Maintaining official branding and distribution channels.
- Deciding whether a contribution fits the local-first product direction.
- Coordinating security reports privately.
- Publishing official releases only after the release checklist passes.

## Becoming a Maintainer

Maintainer access is not granted automatically through contribution volume.

Any future maintainer must be explicitly appointed by Tarun Agarwal and added to
this file and `.github/CODEOWNERS`.
